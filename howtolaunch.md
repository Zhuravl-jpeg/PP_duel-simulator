### 🚀 Инструкция по запуску (согласно `whatswrong.md`)

Скопируйте следующие команды в PowerShell. Они поднимут инфраструктуру и запустят проект.

**1. Запуск PostgreSQL и Redis через Docker:**
```powershell
# PostgreSQL (порт 5432)
docker run -d --name duel-postgres -p 5432:5432 -e POSTGRES_PASSWORD=password -e POSTGRES_DB=duel_reaction postgres:15

# Redis (порт 6379)
docker run -d --name duel-redis -p 6379:6379 redis:alpine
```

**2. Инициализация базы данных:**
```powershell
npm run db:push
```
*Создаст все таблицы (`users`, `matches`, `rounds`, `match_participants`, `round_results`) в локальном PostgreSQL.*

**3. Запуск dev-сервера:**
```powershell
npm run dev
```
*Приложение откроется на `http://localhost:3000`.*

### 🔍 Как проверить работоспособность окружения

1. **БД и Redis:** В терминале запущенного `npm run dev` вы должны увидеть:
   - `[REDIS] Connected successfully` (если Docker поднят)
   - Или `[PROTECTION] REDIS_URL not set, rate limiting disabled` (если Redis не поднят — это нормально, приложение не упадёт).
2. **Auth:** Кнопки входа появятся только после заполнения `GOOGLE_ID`/`GITHUB_ID`. До этого работает вход по Email/Password.
3. **Сборка:** Проверка `npm run build` прошла успешно с новыми `.env` файлами.

Если нужно, могу сгенерировать `docker-compose.yml` для управления контейнерами одной командой или помочь с подключением реальных OAuth-ключей.













### 2. Запуск баз данных (Docker)

Самый надежный способ запустить PostgreSQL и Redis — использовать Docker. Если он у вас установлен, выполните следующие команды в терминале (в папке проекта).

**Шаг А: Запуск PostgreSQL**
```bash
docker run -d \
  --name duel-postgres \
  -p 5432:5432 \
  -e POSTGRES_PASSWORD=password \
  -e POSTGRES_DB=duel_reaction \
  postgres:15
```
*Это создаст контейнер с базой данных и сразу создаст базу `duel_reaction`.*

**Шаг Б: Запуск Redis**
```bash
docker run -d \
  --name duel-redis \
  -p 6379:6379 \
  redis:alpine
```
*Это создаст контейнер с Redis для системы защиты.*








### 3. Инициализация базы данных

Теперь, когда базы данных запущены, нужно «накатить» схему таблиц (users, matches, rounds и т.д.), описанную в Drizzle ORM.

Выполните команду:
```bash
npm run db:push
```
Это создаст все таблицы в вашей PostgreSQL базе. Если всё прошло успешно, вы увидите сообщение об успехе.















### 4. Настройка OAuth (Google / GitHub)

Если вы хотите протестировать вход через GitHub или Google, вам нужно получить «ключи» от этих сервисов. Без этого кнопки входа работать не будут (или будут выдавать ошибки).

**Для GitHub:**
1. Зайдите в **Settings** вашего профиля GitHub -> **Developer settings** -> **OAuth Apps** -> **New OAuth App**.
2. В `Homepage URL` укажите `http://localhost:3000`.
3. В `Authorization callback URL` укажите `http://localhost:3000/api/auth/callback/github`.
4. Скопируйте `Client ID` и `Client Secret` в ваш `.env`.

**Для Google:**
1. Зайдите в **Google Cloud Console**.
2. Создайте проект -> **APIs & Services** -> **Credentials**.
3. Создайте `OAuth client ID` -> Type: `Web application`.
4. В `Authorized redirect URIs` укажите `http://localhost:3000/api/auth/callback/google`.
5. Скопируйте `Client ID` и `Client Secret` в ваш `.env`.

*(Если вы пока не хотите настраивать вход через соцсети, можно просто оставить эти поля пустыми в `.env`, приложение запустится, но кнопок входа не будет).*
